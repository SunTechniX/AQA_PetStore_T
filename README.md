## 🎯 Цель проекта

Научиться писать **надёжные, поддерживаемые и документированные** API-тесты на Python с использованием:
- `requests` — HTTP-клиент
- `pytest` — фреймворк для тестов
- `allure-pytest` — отчётность
- `jsonschema` — валидация ответов
- **Page Object / Client Pattern** — архитектура тестов

**Итог**: Рабочий проект с 24+ тестами, отчётом Allure и интеграцией в CI/CD.

## 🚀 Быстрый старт

### 1. Клонируй репозиторий
```bash
git clone https://github.com/your-username/petstore-api-tests.git
cd petstore-api-tests
```

### 2. Установи зависимости
```bash
# Для РФ — с зеркалом
pip install -r requirements.txt \
    --index-url https://pypi.tuna.tsinghua.edu.cn/simple \
    --trusted-host pypi.tuna.tsinghua.edu.cn
```

### 3. Проверь установку
```bash
pytest --version
flake8 --version
allure --version
```

### 4. Запусти тесты
```bash
# Все тесты
pytest tests/ -v

# Только smoke-тесты
pytest tests/ -m smoke -v

# С отчётом Allure
pytest tests/ --alluredir=allure-results
allure serve allure-results
```

### 5. Отправь на проверку
```bash
git add .
git commit -m "feat: реализованы тесты User API"
git push
```
→ Открой вкладку **Actions** в GitHub — там будет отчёт автопроверки.

## 🚨 Частые ошибки и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| `--timeout` не распознан | Не установлен `pytest-timeout` | `pip install pytest-timeout` |
| `--json-report` не распознан | Не установлен `pytest-json-report` | `pip install pytest-json-report` |
| Тесты падают с 404 | Пет/пользователь не создан | Создавай данные перед тестом |
| Утечка сессий | Не закрыт `client.close()` | Используй фикстуры с `yield` + `close()` |
| Флаки-тесты | Нет ретраев, таймауты | Настрой `Retry` в `base_client.py` |
| Allure не генерирует отчёт | Не указан `--alluredir` | Добавь `--alluredir=allure-results` в `pytest.ini` |

## 💡 Советы для успешной сдачи

1. **Начни с малого**: сначала реализуй `test_pet.py` (4 базовых теста), потом расширяй.
2. **Используй фикстуры**: не создавай клиентов в каждом тесте — вынеси в `conftest.py`.
3. **Валидируй схемы**: это покажет понимание контрактов API, а не только статус-кодов.
4. **Логируй и документируй**: Allure-шаги и `@allure.title` помогут при ревью.
5. **Тестируй локально**: `pytest tests/test_pet.py -v` перед пушем сэкономит время.
6. **Чисти за собой**: удаляй тестовых питомцев/пользователей в `finally` или фикстурах.
7. **Смотри Summary**: после каждого пуша проверяй отчёт — там подсказки по ошибкам.
